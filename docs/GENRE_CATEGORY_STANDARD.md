# 장르 / 카테고리 분류 기준

곡이 수백 곡으로 늘어나도 분류·필터가 일관되게 동작하도록, **장르**와 **카테고리**의 의미와 값 목록을 이 문서로 고정한다.

> 단일 소스: **[`src/js/taxonomy.js`](../src/js/taxonomy.js)**
> 목록을 바꾸려면 이 파일만 고치면 된다. 노래 추가 모달·곡 정보 편집·필터가 모두 여기서 값을 읽는다.

---

## 1. 두 축의 의미

두 축은 **서로 직교**한다. 한 곡은 보통 둘 다 갖는다.

| 축 | 질문 | 뜻 | 예 |
| :--- | :--- | :--- | :--- |
| **장르 (Genre)** | "**무엇처럼 들리는가**" | 음악 스타일 · 사운드 | 락, 힙합, 댄스, 발라드 |
| **카테고리 (Category)** | "**어디 음악인가**" | 씬 · 시장 · 출신 | K-POP, J-POP, 애니메이션, OST |

예시로 보면 명확하다:

- 아이돌 댄스곡 → 장르 `댄스` + 카테고리 `K-POP`
- 일본 애니 오프닝 록 → 장르 `락` + 카테고리 `애니메이션`
- 해외 팝 발라드 → 장르 `발라드` + 카테고리 `POP(해외)`
- 보컬로이드 EDM → 장르 `EDM` + 카테고리 `보컬로이드`

**"K-POP"은 장르가 아니다** — 사운드가 아니라 시장이기 때문이다. 반대로 **"락"은 카테고리가 아니다**.

## 2. 값 목록

저장 값 = 표시 라벨 (한국어). 별도 코드/슬러그를 두지 않는다.

### 장르 (사운드) — 16개
```
발라드, 댄스, 팝, 락, 메탈, 힙합, R&B/소울, 인디,
재즈, EDM, 포크/어쿠스틱, 클래식, 트로트, 펑크, CCM, 기타
```

### 카테고리 (씬/시장) — 11개
```
K-POP, J-POP, POP(해외), 애니메이션, 보컬로이드, 게임,
OST, 뮤지컬, 국악/전통, 라이브/커버, 기타
```

목록에 없는 값이 필요하면 각 드롭다운의 **"직접 입력…"** 으로 커스텀 값을 넣을 수 있다. 커스텀 값은 그대로 저장되고, 다음에 열 때 목록에 `(기존)` 표시로 함께 나온다.

### 경계 사례 판정

| 값 | 판정 | 이유 |
| :--- | :--- | :--- |
| 인디 | **장르** | 사운드/제작 방식의 성격이 강해 장르로 둔다 |
| 트로트 | **장르** | 특정 사운드를 가리킨다 (한국 시장 = 카테고리 `K-POP`과 조합) |
| CCM | **장르** | 사운드/용도 계열 |
| 애니메이션 · 게임 · OST | **카테고리** | 쓰이는 매체(출신)이지 사운드가 아니다 |
| 보컬로이드 | **카테고리** | 씬/제작 형태 |
| 라이브/커버 | **카테고리** | 음원의 형식·출처 |

## 3. 저장 구조 (참고)

- `song.genre` — 문자열 1개 (장르)
- `song.categories` — 배열 (현재 UI는 1개만 넣지만 스키마는 다중 지원)
- `song.curationCategory` — `categories[0]`의 스칼라 미러 (백엔드 `Tracks.curation_category`)

백엔드(`library.rs`)는 `categories`가 오면 그것을 진실의 원천으로 삼아 `Track_Category_Map`과 `curation_category`를 함께 갱신한다.

라이브러리 필터 드롭다운은 **DB에 실제로 쓰인 값**을 읽어 만들므로(`ui/core.js`), 값이 표준을 따르면 필터도 자동으로 깔끔해진다.

## 4. 기존 데이터 재매핑

이 기준을 도입하기 전에는 값이 세 갈래로 섞여 있었다:

1. 예전 UI 프리셋의 **소문자 영어 키** — `kpop`, `ballad`, `rock` …
2. 온라인 메타데이터 자동수집(`seed_tags.json`)이 넣는 **한국어 장르명** — `록`, `인디 록`, `K-팝` …
3. 자유 입력 카테고리 — 제각각

`taxonomy.js`의 `remapClassification()`이 이를 새 기준으로 옮긴다. 앱 첫 실행 시 `main.js`가 **1회만** 전체 라이브러리에 적용하고(`localStorage.taxonomyMigratedV1`), 바뀐 게 있으면 저장한다.

주요 매핑:

| 예전 값 | → 새 값 |
| :--- | :--- |
| `rock`, `록`, `팝 록`, `얼터너티브 록` | 장르 `락` |
| `hiphop`, `랩` | 장르 `힙합` |
| `rnb`, `알앤비`, `소울` | 장르 `R&B/소울` |
| `indie`, `인디 록` | 장르 `인디` |
| `edm`, `일렉트로닉` | 장르 `EDM` |
| `folk`, `어쿠스틱` | 장르 `포크/어쿠스틱` |
| **`kpop`, `K-팝`** | **카테고리 `K-POP`** (장르 → 카테고리로 이동) |
| **`jpop`, `제이팝`** | **카테고리 `J-POP`** |
| **`anime`, `애니송`** | **카테고리 `애니메이션`** |
| **`vocaloid`, `ost`, `game`** | **카테고리 `보컬로이드` / `OST` / `게임`** |

핵심은 **예전에 장르 자리에 있던 씬 값(kpop 등)이 카테고리로 옮겨간다**는 점이다. 매핑표에 없는 값은 손대지 않고 장르에 그대로 보존한다(사용자 커스텀 값 보호).

## 5. 알려진 후속 과제

- `src-tauri/src/seed_tags.json`의 자동수집 taxonomy는 여전히 자체 한국어 계층(`록` → `인디 록` 등)을 쓴다. 현재는 `remapClassification()`이 저장 시점에 흡수하지만, 근본적으로는 자동수집 쪽도 이 표준을 직접 내보내도록 정리하는 게 좋다.
- `index.html`의 라이브러리 툴바 `#lib-genre-dropdown`에 남아 있는 초기 하드코딩 옵션은 로드 직후 JS가 DB 값으로 덮어쓰므로 실사용에는 영향이 없다(플래시 방지 차원의 정리 대상).
